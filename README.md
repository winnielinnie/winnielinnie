# Winston

I work on developer platforms, serverless, and infrastructure products. I spend most of my time in product and technical strategy, but I still like writing code when there is a real workflow worth shaving down.

Most of the projects here are small on purpose. I would rather publish a tool that does one job cleanly than a big demo repo that nobody actually uses.

A few places to start:
- [site](https://winnielinnie.github.io/winstonlin-site/) — writing, case studies, and selected projects
- [`oci-fn-object-storage-router`](https://github.com/winnielinnie/oci-fn-object-storage-router) — a practical OCI Functions pattern
- [`release-note-diff-cli`](https://github.com/winnielinnie/release-note-diff-cli) — a tiny CLI for finding meaningful release note changes faster

Things I tend to care about:
- developer experience
- migration and adoption pain
- event-driven systems
- pragmatic Python tooling
- making operational work easier to reason about

I am usually more interested in useful than impressive.
